## 📅 15.03.2026 (воскресенье)

### ✅ Сделано

#### ⚡ TimePicker — полный реворк производительности
- `ScrollView` заменён на `FlatList` с `getItemLayout` — рендерится только видимое окно (~10 элементов)
- Множитель уменьшен с `×200` до `×40` (итого 960 и 2400 элементов против 2 880 и 12 000)
- Убран `initialScrollIndex` — заменён на `onLayout` + `scrollToOffset` с точным пикселем
- Добавлен `disableIntervalMomentum={true}` — список всегда останавливается строго на ближайшей позиции
- Результат: открытие мгновенное, защёлкивание работает корректно
- ⚠️ Небольшие шероховатости при очень быстром броске — оставлено в техдолг, не критично

#### 📅 DatePicker — нативный
- Заменён кастомный компонент на `react-native-modal-datetime-picker`
- Вызывает системный Android DatePicker с локализацией `ru_RU`
- Кнопка ✕ для сброса deadline через проп `allowClear`

#### 🔄 Фикс deadline у цикличных задач
- Раньше: `deadline = newTask.deadline || newTask.date` → дата начала всегда уходила как дедлайн
- Теперь: `deadline: newTask.deadline || null` → сервер получает `null` если не выбрано
- Следующая итерация цикличной задачи больше не получает «случайный» дедлайн

#### 🗂️ Доп. настройки скрыты по умолчанию
- `showAdvancedSettings = false` при открытии новой задачи через FAB
- При редактировании — раскрываются автоматически только если есть время / deadline / повторение

#### 📜 Фикс скролла в модалке
- Проблема: `TouchableOpacity` / `TouchableWithoutFeedback` перехватывали gesture-события у вложенных ScrollView/FlatList
- Решение v1: `TouchableWithoutFeedback` — помогло частично (первый скролл работал, повторные лагали)
- Решение v2 (финальное): `Pressable` — не захватывает responder у дочерних компонентов, скролл стабилен
- `nestedScrollEnabled` на ScrollView модалки для надёжности

#### 🗂️ Этап 2: Папки — полная реализация
- API-слой: `foldersAPI` (get/create/update/delete/reorder) — всё в `TasksScreen.js`
- UI: горизонтальный ScrollView чипов. Тап = фильтр, долгое нажатие = меню действий
- Меню папки: переместить влево/вправо, редактировать, удалить
- Drag-and-drop: `PanGestureHandler` + `LongPressGestureHandler` (pan снаружи, longPress внутри)
- Призрак задачи под пальцем во время drag
- Анимированный пульс у папок-зон при наведении (`FolderDropChip` с `Animated.loop`)
- Выбор папки при создании/редактировании задачи в раскрывашке «Дополнительно»
- `folderId` корректно передаётся на сервер через все CRUD-операции
- Toast-уведомления при всех ключевых действиях

#### 🐛 Багфиксы в рамках Этапа 2
- **Drop не работал** — координаты чипов не успевали записаться до первого drag. Решение: `setTimeout(..., 50)` + перезамер `foldersRowRef` в момент старта drag
- **Зона попадания** расширена с ±10px до ±20px по вертикали
- **Анимация hover** была через `transform.scale` в state — теперь отдельный `FolderDropChip` с собственным `Animated.Value` и `Animated.loop`

### 🐛 Известные проблемы / Техдолг
- TimePicker: небольшой дребезг при очень быстром броске (не критично, оставлено на потом)
- Modal scroll: изредка чуть медленнее при первом касании на старых Android (наблюдаем)
